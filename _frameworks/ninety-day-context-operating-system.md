The operating system is not a transformation program. It is a sequence of first containers.

## Sequence

- **Days 1-14:** map value and name the pain.
- **Days 15-30:** clarify recurring decision rights and canonical records.
- **Days 31-45:** create one conflict path and repair loop.
- **Days 46-60:** protect one AI-exposed formation ladder.
- **Days 61-90:** install a review cadence and ask what changed, what held, and what still leaks.

Temporal architecture supports the whole sequence: maps, decisions, conflicts, and formation rungs only survive when someone owns the rhythm of review.
